WP_PROXY_HOiST拯救你的wordpress429

发表于 2019-11-17 更新于 2021-07-13 分类于 WordPress Waline： 阅读次数：

众所周知，wordpress429已多日，在网上搜索此情况，各路dalao纷纷只是给个原因，或者如何手动更新wordpress,更有写出了个插件更新wordpress主程序的，但是，这些是完全不能满足我们的，Wordpress本身就有数不胜数的优秀插件，然鹅，遇到这个429，我们就只能放弃更新插件，转移服务器到境外或者只能手动更新插件嘛，这也太麻烦了趴，于是，我开始在wordpress官网文档上摸索和，通过借鉴一些大佬的教程，我发现了WP_PROXY_HOST但是，我按照他们的的代码复制粘贴，如何，貌似并没用啊，或许是他们的服务器已经关掉了

在官网上，专门有对WP_PROXY_HOST的介绍

大致就是说wordpress提供了http代理的功能，嗯，就这样

我们只需增加如下代码到wp-config.php后重载nginx即可激活http代理功能

/* 配置代理 */
define('WP_PROXY_HOST', '127.0.0.1');//代理服务器
define('WP_PROXY_PORT', '8118');//代理端口
define('WP_PROXY_USERNAME', '');//代理用户名（可选）
define('WP_PROXY_PASSWORD', '');//代理密码(可选）
define('WP_PROXY_BYPASS_HOSTS', 'localhost');//排除域名

问题来了，我http代理呢！    这个问题很简单，通过搜索引擎，就可以很快学会使用国外机子搭建一个http代理（记得限制使用IP），或者，可以去国外找http代理（但是由于gfw原因，不能保证能用，而且不一定是给你wordpress用的，或许他个类似只放行给tg这种代理，况且有安全隐患）

上述只是分享一些思路，本站不提供http代理，我自认为比那些教…算了   大伙看了我思路自己去折腾趴（两行代码扯了一大堆，逃~

还是他妈跪舔iOS，iOS13加入了深色模式，贵司就马不停蹄赶紧更新，生怕更的晚了赶不上库克的临幸，等到Android的时候就说什么“我们不加入深色模式，是想让你多休息”，你有本事跟12306一样，23:00-6:00关服务器，全部下线啊。双重又当又立马化腾和张小龙亲妈是真的忙开了花。
